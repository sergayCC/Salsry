# Salary Ticker (GitHub Pages)

這是一個純前端、可直接部署到 GitHub Pages 的薪資即時計算工具。

## 功能

- 輸入月薪、每日工時、上班開始與結束時間
- 依台灣工作日規則（排除週六日與國定假日）計算
- 顯示秒薪、分薪、時薪、日薪
- 顯示今日已賺、當月已賺、當年已賺
- 每秒更新畫面

## 專案檔案

- `index.html`: 頁面結構
- `styles.css`: 視覺樣式
- `app.js`: 計算邏輯與互動
- `data/tw-holidays-2026-2030.json`: 假日資料檔（可自行維護）

## 假日資料 Schema

假日資料檔為 JSON 物件，年份是 key、value 是日期字串陣列。

```json
{
  "2026": ["2026-01-01", "2026-02-16"],
  "2027": ["2027-01-01"],
  "2028": [],
  "2029": [],
  "2030": []
}
```

規則：

- 年份 key 必須是 4 位數，例如 `2031`
- 日期格式必須是 `YYYY-MM-DD`
- 日期必須落在對應年份（例如 `2031-...` 只能放在 `2031`）
- 同一年不可重複日期

## 年度維護 SOP（自行更新假日）

1. 開啟 `data/tw-holidays-2026-2030.json`
2. 新增下一年 key（例如 `2031`），填入該年國定假日日期
3. 儲存後直接開啟頁面測試
4. 檢查畫面是否正常顯示數值，且無錯誤訊息
5. 推送到 GitHub 後，確認 GitHub Pages 線上頁面也正常

提醒：如果資料格式錯誤，頁面會顯示可讀錯誤提示，請依提示修正。

## 本機開啟

可直接用瀏覽器開啟 `index.html`。若瀏覽器限制本機 `fetch` 讀檔，請改用簡易靜態伺服器（例如 VS Code Live Server）。

## GitHub Pages 部署

1. 將專案推到 GitHub repository
2. 到 repo 的 `Settings > Pages`
3. Source 選 `Deploy from a branch`
4. Branch 選 `main`（或你的主要分支）和 `/ (root)`
5. 儲存後等待部署完成
